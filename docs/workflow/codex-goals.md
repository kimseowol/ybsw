# Codex Goal 운용 — 1차 프로토 전체 구현

승인된 PRD 7종의 전체 구현을 Codex goal 4개의 연쇄로 진행한다.
근거: goal은 "결과 + 검증 표면"이 선명할 때만 자동 계속(auto-continue)이 안전하다 —
"전부 다"류의 광범위 goal은 감사 표면이 없어 안티패턴이다(OpenAI cookbook).

## 운용 규칙

- **headless 운용**: Claude가 `codex exec`로 goal을 프롬프트로 주입해 구동한다. 대화형 auto-continue 대신 Claude가 계속/재개(`codex exec resume`)를 판단하는 오케스트레이터 역할을 한다. 대화형으로 쓸 경우엔 아래 텍스트를 `/goal `에 붙여넣는다 (codex 0.128.0+).
- **headless는 `--dangerously-bypass-approvals-and-sandbox` 필수**: codex exec의 업스트림 결함(openai/codex#24135)으로 MCP 도구 승인이 headless에서 자동 거부된다("user cancelled MCP tool call"). 우회 config 키가 없어 이 플래그가 유일한 경로다 — 2026-07-07 사용자 승인, 로컬 신뢰 프로젝트 한정. 결함 수정 시 플래그를 제거한다.
- **한 마일스톤 = 한 세션 = 한 goal.** — 개정 2026-07-07 (사용자): Goal 1이 실측 20분에 완료돼 세션 분할 오버헤드가 이득을 넘어섰다. Goal 2~4는 **한 세션·한 goal로 통합 실행**하되, 마일스톤별 검증 표면은 내부 게이트로 유지하고(이전 마일스톤 표면 전부 그린 + 커밋 후 다음 착수) 마일스톤 완료마다 중간 보고를 남긴다.
- goal이 완료를 보고하면 **Claude 리뷰 게이트**를 거친다 — 접합부 계약·PRD 경계 준수·테스트 실질성(단언이 진짜를 재는지)을 검사한 뒤 다음 goal을 활성화한다.
- 규범의 원본은 goal이 아니라 **AGENTS.md + 승인 PRD**다. goal은 범위와 완료 증거만 정의한다. 충돌 시 PRD가 이긴다.
- goal에 "PRD가 없으면 기술 PRD를 만든다" 같은 라우팅 규범을 복제하지 않는다. PRD 생성 여부는 실행 시 `docs/workflow/prd.md` 0절에서 판정하고, 동작 불변 구조 변경은 goal 안에서도 새 PRD의 근거가 되지 않는다.
- 모든 goal 공통: 결정 기록 실시간 append(AGENTS.md 규칙 3), PRD 상태·INDEX 갱신, 완료 보고에 검증 내역 포함(침묵 통과 금지).

## Goal 1 — 기반 계층 + 첫 플레이어블

```
승인된 PRD-001·PRD-002 전체, PRD-005의 이동·평타, PRD-006의 최소 스폰, PRD-007의 시나리오 러너 골격으로 "첫 플레이어블"을 완성한다.
결과: 게임 씬에서 캐릭터가 이동하고, 몬스터가 스폰되어 추적하며, 평타가 자동 발사되고, 피해→죽음→풀 반환→XP 지급이 전 계층을 관통하며, 씬 리로드로 재시작된다.
착수 순서: 접합부 골격을 먼저 굳힌다 — asmdef 의존 방향(PRD-001 경계), 씬 스코프 컴포지션 루트, 큐잉 이벤트 디스패처(등록순·엔벨로프·프레임 상한), 피해·회복 파이프라인(HP 변이 봉인), 액터·진영 쿼리, 입력 시임, 시나리오 데이터 에셋+러너+스크립트 입력 시퀀스 템플릿 1개. 그 다음 첫 플레이어블 콘텐츠를 그 위에 얹는다.
검증 표면: EditMode 계약 테스트(등록순 디스패치, 이벤트 재귀 금지, HP 우회 불가, 죽음 1회 확정, 파이프라인 단계 순서) 그린 + 시드 고정 PlayMode 시나리오 1개 이상(예: 60초 내 킬 ≥ N, 음수 HP·중복 죽음 0건) 그린 + 컴파일 클린 + scripts/check-prd.ps1 통과 + 100체 이상 스폰 시 프레임 타임 기록.
회귀 금지: PRD 경계 위반(HP 단일 경로, SO 런타임 읽기 전용, 봇 전용 입력 경로 신설, 정적 전역 상태) 금지. docs/design/ 수정 금지, --no-verify 금지.
반복 정책: 매 반복마다 컴파일 확인→관련 테스트→시나리오 실행 순으로 검증하고, 개발 측 판단이 필요했던 결정은 해당 PRD 결정 기록에 그 자리에서 append한다. 시나리오·테스트는 지우지 말고 축적한다.
중단 조건: 기획자 확인이 필요한 사항이 나오거나, PRD 경계 자체를 고쳐야 진행 가능해 보이거나, Unity 에디터 연결이 복구 불가면 — 시도한 경로·증거·블로커·필요한 입력을 정리해 멈추고 보고한다.
```

## Goal 2 — 쫄 시스템

```
승인된 PRD-004 전체를 구현한다: 쫄따구가 캐릭터를 추종하며 짱돌 투사체로 몬스터를 공격하고, 스킬 소환수(해골)와 공유하는 아군 유닛 기반 위에 서며, 파생 스탯은 본체 스탯 변경 이벤트에 라이브 재계산된다.
결과: 시드 고정 시나리오에서 쫄 유무에 따른 킬 수 차이가 단언되고, 본체 스탯 상승 시 이미 소환된 개체의 스탯이 갱신됨이 단언된다. 개체 생성·회수는 풀 경유, 처치 귀속은 소유자(플레이어).
검증 표면: Goal 1의 전체 테스트 그린 유지 + 쫄 시나리오(추종 이탈 없음, 쫄 DPS 기여 > 0, 라이브 파생 재계산) 그린 + check-prd.ps1 통과. 수치 없는 스펙(짱돌·추종 간격)은 장르 관례 시작값을 데이터 에셋으로 두고 결정 기록에 남긴다.
회귀 금지: 쫄 행동을 스킬 훅에 넣지 않는다(PRD-003 경계). 몬스터는 플레이어만 공격 — 쫄은 피격 비대상(PRD-006 결정).
반복 정책·중단 조건: Goal 1과 동일.
```

```
승인된 PRD-003 전체와 PRD-005의 잔여(전직·E/R)를 구현한다: 정의(SO)/영속 런타임/시전 인스턴스 3계층, 트리거·타겟팅·타임라인, 이펙트·큐 리스트, 제한된 커스텀 훅 슬롯, Modifier 3층(개별·태그·전역, 합연산+희귀 곱연산 그룹), 그리고 GD-003 콘텐츠 — 평타 스킬화, 전직(1분 후 고정 몬스터 처치, 전부 유지+E/R 추가), Icicle·Ice Age(이동하며 즉시 발동), 자동 발동 스킬 5종(단도 투척·낙뢰·해골 디버프·해골 소환·태불망), 레벨 1~9 보너스, 상태이상(기절·둔화·디버프).

반복 정책·중단 조건: Goal 1과 동일.
```

## Goal 1 리뷰 이월 사항 (Goal 2~4에서 소화)

Goal 1 리뷰 게이트는 통과(PASS)했고, 아래 3건은 결함이 아니라 "해당 기능이 실제로 쓰이기 시작하는 시점에 함께 넣을 것"으로 이월됐다.

1. **디스패처 구독 해지 컴팩션** — `GameplayEventDispatcher`의 unsubscribe가 soft-delete로만 남는다. 쫄·소환수처럼 개체 단위 구독이 대량 생성·소멸되는 시점(쫄 시스템)에 드레인 후 컴팩션을 넣는다.
2. **ChainDepth 상한 강제** — `EventEnvelope.ChainDepth`가 운반만 되고 상한이 강제되지 않는다. 처치 반응·디버프 등 반응형 효과 콘텐츠(스킬 시스템)와 함께 상한 강제 + 테스트를 넣는다.
3. **인터셉터 스냅샷 순회** — `DamagePipeline` 인터셉터 목록이 순회 중 등록·해지에 취약하다. 인터셉터를 실제로 쓰기 시작할 때(스킬 시스템) 스냅샷 순회로 바꾼다.

## Claude 리뷰 게이트에서 보는 것

1. **접합부 계약** (Goal 1 종료 시 가장 중요) — asmdef 의존 방향, HP 봉인, 디스패처 순서 계약이 실제 코드로 강제되는가.
2. **경계 준수** — PRD "지켜야 할 것"의 표본 검사 + 우회 경로 탐색.
3. **테스트 실질성** — 단언이 실제 동작을 재는가, 통과를 위한 형식 테스트는 아닌가.
4. **결정 기록** — 개발 판단이 PRD에 append됐는가, 기획 체감에 걸치는 항목이 보고됐는가.
