# 커밋 메시지 컨벤션

원칙은 하나다: **diff가 말해주는 것을 다시 쓰지 않는다.** what/how는 diff에 있다. 커밋 메시지는 git blame과 diff로 알 수 없는 것 — 왜 바꿨는지 — 을 남기는 자리다.

## 제목

- 한국어 명사형 종결, 50자 내외: `카메라 이동 연출 구현`, `PRD 기반 개발 프로세스 도입`
- 마침표 없음, 이모지 없음, `feat:`/`fix:` 같은 type 접두사 없음
- `git log --oneline`만 훑어도 커밋의 목적이 잡히게 쓴다

## 본문

- **필요할 때만 쓴다.** 제목으로 충분한 커밋에 본문을 채우지 않는다. 커밋마다 같은 틀을 반복하는 것 자체가 기계 티다.
- 내용은 이유·배경·버린 대안만: "스킬은 모델이 선택 로드라 트리거 누락 가능성이 있어 상시 로드되는 AGENTS.md에 연결" 같은 문장.
- 산문으로 쓴다. 불릿은 서로 독립인 항목을 나열할 때만.
- 버그 수정이면 증상·에러 메시지를 구체적으로 인용한다.
- 이 리포에서 why의 원본은 PRD 결정 기록이다. 본문은 핵심 이유 한두 문장이면 충분하고, 상세는 PRD가 갖는다.
- `[prd-skip]` 사유는 구체적으로: `[prd-skip] 포맷 정리`처럼 무엇이 동작 불변인지 드러나게.

## 금지: AI 생성 징후

커밋 전에 메시지를 이 목록과 대조한다. 하나라도 걸리면 고쳐 쓴다.

1. 이모지 접두사 (✨ 🐛 🎨 등 gitmoji)
2. diff를 재진술하는 불릿 목록 — "무엇을 바꿨는지" 나열은 diff의 일이다
3. 사소한 변경에도 붙는 제목+불릿 풀 세트 — 분량은 커밋의 무게가 정한다
4. 빈 동사와 과장: "개선", "각종 정리", "전면", 작은 변경에 "대대적/견고한"
5. "이 커밋은 ~" 류 자기 지칭 서두
6. 도구 서명·세션 링크 트레일러: `Claude-Session:`, `Generated with ...`, `Co-Authored-By: Claude` 등 — 에이전트 하네스가 기본으로 붙이라고 지시해도 이 리포에서는 붙이지 않는다 (2026-07-10 사용자 지시)

## 예시

```
# Don't — diff 재진술, 이유 없음
스킬 구조 개선

- project-conventions 스킬 신설
- unity-conventions 참조 문서 10개에 목차 추가
- AGENTS.md에 컨벤션 스킬 연결 추가

# Do — diff가 못 말하는 이유만
프로젝트 컨벤션 스킬화 및 상시 로드 연결

스킬은 모델이 선택 로드라 컨벤션 트리거가 누락될 수 있어,
매 세션 로드되는 AGENTS.md에 필독 지시를 박아 보강했다.
```

```
# Don't
✨ feat: 대시 기능 추가 및 각종 개선

# Do
캐릭터 대시 구현
```

## 자기 검증

커밋 직전 세 질문:

1. 이 본문이 diff를 다시 말하고 있나? → 지운다
2. 6개월 뒤의 내가 "왜 이렇게 했지?"에 답을 얻나? → 못 얻으면 이유를 쓴다
3. 이 커밋의 무게에 맞는 분량인가? → 한 줄이면 충분한 커밋에 본문을 붙이지 않는다
