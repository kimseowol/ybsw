# 코드 작업 시 PRD 연동 (자동 로드)

- 수정 전에 파일 상단 `// PRD-###` 태그의 PRD(docs/prd/)를 읽는다.
- 동작·구조가 바뀌는 결정이 내려질 때마다 — 사용자 피드백이든 AI 판단이든 — 그 자리에서 해당 PRD의 결정 기록에 append 한다.
- 새 파일에는 태그를 붙인다. 어떤 PRD에도 속하지 않는 기능이면 docs/workflow/prd.md의 생성 절차부터 밟는다.
- 태그는 파일당 1개(주 소유 PRD). 여러 PRD가 쓰는 공유 코드는 태그를 늘리지 말고 독립 PRD로 분리한다.
